第二章：核心数据表

sys_user（用户表）

CREATE TABLE `sys_user` ( `user_id` BIGINT NOT NULL COMMENT '用户ID', `username` VARCHAR(50) COMMENT '用户名', `password` VARCHAR(100) COMMENT '密码', `salt` VARCHAR(100) COMMENT '随机盐', `phone` VARCHAR(20) COMMENT '手机号', `dept_id` BIGINT COMMENT '部门ID', `lock_flag` CHAR(1) DEFAULT '0' COMMENT '锁定标记', `del_flag` CHAR(1) DEFAULT '0' COMMENT '删除标记', PRIMARY KEY (`user_id`), UNIQUE KEY `uk_username` (`username`) ) COMMENT '用户表';

sys_role（角色表）

CREATE TABLE `sys_role` ( `role_id` BIGINT NOT NULL COMMENT '角色ID', `role_name` VARCHAR(50) COMMENT '角色名称', `role_code` VARCHAR(50) COMMENT '角色编码', `role_desc` VARCHAR(255) COMMENT '角色描述', `del_flag` CHAR(1) DEFAULT '0', PRIMARY KEY (`role_id`) ) COMMENT '角色表';

sys_menu（菜单权限表）

CREATE TABLE `sys_menu` ( `menu_id` BIGINT NOT NULL COMMENT '菜单ID', `name` VARCHAR(50) COMMENT '菜单名称', `permission` VARCHAR(100) COMMENT '权限标识', `path` VARCHAR(255) COMMENT '路由路径', `parent_id` BIGINT COMMENT '父菜单ID', `type` CHAR(1) COMMENT '0-菜单 1-按钮', PRIMARY KEY (`menu_id`) ) COMMENT '菜单权限表';

第四章：Redis 缓存设计

缓存 Key 规范

Key 前缀 用途 TTL 示例
pig_oauth_access: OAuth2 Access Token 7200s pig_oauth_access:eyJhbG...
pig_oauth_refresh: Refresh Token 7天 pig_oauth_refresh:xxx
user_details: 用户详情缓存 30分钟 user_details:admin
sys_menu: 菜单缓存 30分钟 sys_menu:*
captcha_code: 验证码 5分钟 captcha_code:uuid

Token 存储结构

🔑 Access Token

Key: pig_oauth_access:{token}
Value: OAuth2Authorization JSON
{
  "id": "授权ID",
  "principalName": "用户名",
  "accessToken": {
    "tokenValue": "xxx",
    "expiresAt": "12:00:00"
  }
}
TTL: 7200s (2小时)

🔄 Refresh Token

Key: pig_oauth_refresh:{token}
Value: OAuth2Authorization JSON
{
  "id": "授权ID",
  "refreshToken": {
    "tokenValue": "xxx",
    "expiresAt": "+7天"
  }
}
TTL: 604800s (7天)

第六章：性能优化

索引优化建议

  • 用户名索引：已存在唯一索引 uk_username
  • 部门树查询：添加 idx_dept_parent ON sys_dept(parent_id)
  • 日志查询：按时间分区 + idx_log_create_time
  • 角色菜单：联合索引 idx_role_menu(role_id, menu_id)

查询优化清单

  • 使用索引覆盖，避免 SELECT *
  • 批量查询替代循环单条查询
  • 使用 MyBatis Plus 分页插件
  • 避免 N+1 查询问题
  • 大表考虑分区或分表

第七章：安全配置

⚠️ 安全提醒

生产环境务必遵循最小权限原则，禁止使用 root 账户连接数据库。

MySQL 用户权限

-- 创建应用用户（最小权限） CREATE USER 'pig_app'@'%' IDENTIFIED BY 'secure_password'; -- 仅授予必要权限 GRANT SELECT, INSERT, UPDATE, DELETE ON pig.* TO 'pig_app'@'%'; -- 禁止危险操作 REVOKE DROP, ALTER, CREATE ON pig.* FROM 'pig_app'@'%';

Redis 安全配置

# redis.conf bind 127.0.0.1 10.0.0.1 # 绑定内网地址 requirepass your_secure_password rename-command FLUSHALL "" # 禁用危险命令 rename-command KEYS "" # 禁用 KEYS 命令
