第三章：Nacos 注册中心

3.1 为什么需要 Nacos？

📞 类比理解：
没有 Nacos：你想找张三，但你不知道他在哪，你得记住他的手机号、地址。张三搬家了，你就找不到他了。

有 Nacos：你想找张三，查电话簿（Nacos）。电话簿实时更新张三的联系方式，即使张三搬家了，电话簿也会更新。

3.2 服务注册流程

1

服务启动

pig-auth 服务启动，读取配置文件中的 Nacos 地址

2

发送注册请求

向 Nacos 发送：服务名=pig-auth, IP=192.168.1.100, 端口=3000

3

心跳保活

每 5 秒发送一次心跳，告诉 Nacos "我还活着"

4

服务发现

其他服务查询 "pig-auth" 时，Nacos 返回可用实例列表

第五章：服务间通信（Feign）

5.1 什么是 Feign？

类比理解：Feign 让你像调用本地方法一样调用远程服务，不需要关心 HTTP 请求细节。 
// 定义 Feign 客户端接口
@FeignClient(name = "pig-upms")  // 只需要服务名
public interface RemoteUserService {
    @GetMapping("/user/{id}")
    R<UserDTO> getUser(@PathVariable("id") Long id);
}

// 使用时像调用本地方法一样
@Autowired
private RemoteUserService remoteUserService;

public void doSomething() {
    R<UserDTO> result = remoteUserService.getUser(1L);  // 自动负载均衡
}

RBAC = Role-Based Access Control（基于角色的访问控制）

类比理解：公司组织架构 - 用户 → 角色 → 权限

6.3 代码中的权限检查

@RestController
@RequestMapping("/user")
public class SysUserController {

    @GetMapping("/page")
    @HasPermission("sys_user_view")  // 权限检查
    public R<Page> getUserPage() {
        return userService.getPage();
    }

    @DeleteMapping
    @HasPermission("sys_user_del")  // 删除权限
    public R userDel(Long[] ids) {
        return userService.removeByIds(ids);
    }
}

第七章：统一响应格式

7.1 R 响应对象

// 所有接口返回统一格式
public class R<T> {
    private int code;       // 状态码：0=成功，1=失败
    private String msg;     // 提示信息
    private T data;         // 数据内容

    // 成功响应
    public static <T> R<T> ok(T data) {
        R<T> r = new R<>();
        r.setCode(0);
        r.setMsg("success");
        r.setData(data);
        return r;
    }

    // 失败响应
    public static <T> R<T> failed(String msg) {
        R<T> r = new R<>();
        r.setCode(1);
        r.setMsg(msg);
        return r;
    }
}

7.2 响应示例

// 成功响应
{
    "code": 0,
    "msg": "success",
    "data": {
        "userId": 1,
        "username": "admin",
        "nickname": "管理员"
    }
}

// 失败响应
{
    "code": 1,
    "msg": "用户名或密码错误",
    "data": null
}

第九章：常见问题 FAQ

Nacos 和 Eureka 有什么区别？

特性 Nacos Eureka
配置中心 ✅ 支持 ❌ 不支持
健康检查 TCP/HTTP/MySQL 仅 HTTP
社区活跃度 阿里维护，活跃 Netflix 停止维护

@Inner 注解是什么意思？

@Inner 表示这个接口只允许内部服务调用：

  • 外部请求（浏览器）不能直接调用
  • 只有 Feign 内部调用（带有 from=Y 请求头）才能访问
  • 防止外部直接绕过网关调用内部接口

为什么要用 Redis 存储 Token？

  • 性能：Redis 是内存数据库，读取速度极快
  • 过期：Redis 支持 TTL，Token 过期自动删除
  • 分布式：多个服务实例可以共享 Token 状态
